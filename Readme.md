# BubblesOnChunkGen

A Paper plugin that activates bubble columns in rivers generated by Terra's CHIMERA world generator preset. CHIMERA's preset places blue concrete as a marker at the bottom of rivers where bubble columns should appear; this plugin converts those markers into working soul sand bubble columns at runtime.

## How it works

### New chunk generation

When a new chunk is generated, the plugin does two things:

1. **Immediately** (in the chunk load event itself): scans for blue concrete markers, finds the water surface above each one, registers those surface coordinates for water flow prevention, and places a level-3 water block at the surface for a visual stepping effect.

2. **After a short delay** (5 ticks): replaces each blue concrete block with soul sand and places a bedrock block directly below it. The bedrock serves as a permanent signature so the plugin can re-identify these columns when the chunk is loaded again later.

The two-phase approach is necessary because block replacements need all generation stages to finish, but water flow prevention must be active before physics can tick.

### Existing chunk loading

When a previously generated chunk is loaded, the plugin scans for the bedrock signature (starting from Y=49 up to Y=165). For each bedrock block found:

- If the soul sand above has been removed (e.g. by a player), it is automatically restored. This prevents players from griefing bubble columns, which would allow water to flow freely and cause river expansion lag.
- The water surface above the column is located and registered for flow prevention, and a level-3 water block is placed there.

### Water flow prevention

Bubble columns push water upward to the surface. Without intervention, water at the surface flows sideways, creating new source blocks and causing rivers to grow uncontrollably. This is both visually wrong and a source of server lag from cascading water physics.

The plugin uses Bukkit's `BlockFromToEvent` to cancel water flow at the surface coordinates above each bubble column. Blocked coordinates are tracked in a `HashSet` for O(1) lookups and are cleaned up when chunks unload. This approach is invisible to players (unlike barrier blocks) and does not interfere with player movement or boat travel.

### Level-3 water at the surface

A shallow (level 3) water block is placed at the surface position just above each bubble column. Because the `BlockFromToEvent` handler prevents it from flowing, it stays in place as a partial-height water block. This creates a natural-looking visual step at the river surface where the bubble column meets the top of the water, rather than an abrupt air-to-water transition.

### Bedrock below soul sand

A bedrock block is placed directly below each soul sand block for two reasons:

1. **Identification**: it distinguishes plugin-placed soul sand from player-placed soul sand, so the plugin only manages its own columns.
2. **Visibility**: unlike barrier blocks, bedrock is clearly visible to players. If a player with operator permissions chooses to remove it, they are making a conscious decision and accepting the water flow consequences.

### Soul sand restoration on chunk load

Each time a chunk loads, the plugin checks that soul sand is still present above every bedrock signature block. If the soul sand has been broken or removed, it is silently replaced. This ensures bubble columns survive griefing and continue to function as intended without manual intervention.

## Commands

- `/bubblesdebug` - Toggles debug logging on/off (op-only). When enabled, logs to console:
  - Every new chunk detected (coordinates)
  - Number of soul sand blocks placed per chunk
  - Number of surface blocks registered for flow prevention
  - Number of soul sand blocks restored on chunk load
  - Individual surface block coordinates being protected

## Easter egg

There is a 1-in-1000 chance per bubble column that a dedication chest is placed on an adjacent block. The chest contains a written book, emeralds, and diamonds. Chests are only placed on air, water, or cave air blocks that are not buried under a solid block above.

## Requirements

- Paper 1.21+
- Terra (with a configuration that places blue concrete markers for bubble columns)
